Name

tiny.py

Synopsis

For classes that are mostly data stores, with few (or no) methods.

Install

  • From http://git.io/tinytimm download tiny.py. Test as follows: python3 tiny.py; echo $?
  • This should report "0" (i.e. no errors found).

Description

After watching "Stop Writing Classes", https://www.youtube.com/watch?v=o9pEzgHorH0, I tried replacing all my classes with lots of dictionaries (with the set/get # trick below, line). Worked a treat! Code shrank by a factor of two. Still, sometimes, I do want to add a method or two, maybe use Python's magic methods. So...

Examples

Import code

from tiny import so,do

Define a class Robot under class o with three default attributes

so(Robot=o, model=1970, n=2, hi=100)

Class definitions can be any number of attributes, and even include one line methods:

so(Robot=o, model=1970, n=2, hi=100, __lt__=lambda i, j: i.model < j.model)

For another way to add methods, or for longer methods, use the @do decorator.

 @do(Robot)
 def __lt__(i, j): return i.model < j.model

If used with no arguments, the do decorator remembers the last class defied by so or used in a do (in this case, Robot).

 @do()
 def __gt__(i, j): return i.model > j.model

After that, everything works like a normal class:

 assert str(Robot()) == "Robot{'hi': 100, 'model': 1970, 'n': 2}"
 r = Robot(model=34) # can create an instance, override defaults
 assert r.n == 2     # will build with local defaults
 r.n = 49            # can set variables
 assert r.n == 49    # let me show you
 assert str(r) == "Robot{'hi': 100, 'model': 34, 'n': 49}"
 assert r < Robot()  # can meta
